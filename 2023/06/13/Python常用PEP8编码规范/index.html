Python常用PEP8编码规范

PEP8：Style Guide for Python Code

一、缩进
  • 每级缩进使用4个空格
  • 括号中使用垂直隐式缩进或使用悬挂缩进
二、最大行宽
  • 每行最大行款不超过79个字符
  • 一般续行可以使用反斜杠
  • 括号内续行不需要使用反斜杠
三、空行
  • 两行空行用于分割顶层函数和类的定义
  • 单个空行用于分割类定义中的方法
四、模块导入
  • 导入的每个模块应该单独成行
  • 导入顺序（各模块类型导入之间要有空行分割，各组里面的模块的顺序按模块首字母自上而下升序排列）
    • 标准库
    • 相关的第三方库
    • 本地库
五、缩进
  • 使用四个空格来缩进代码
  • 不要使用 tab ，也不要 tab 和空格混用
六、空格
  • 按照标准的排版规范来使用标点两边的空格
  • 括号内不要有空格
  • 不要在逗号，分号，冒号前面加空格，但应该在它们之后加（除了在行尾）
  • 参数列表、索引或切片的左括号前不应该加空格
  • 赋值等操作符前后不能因为对齐而添加多个空格
  • 二元运算符两边各放置一个空格
    • 涉及 = 的符合操作符（+= -= 等）
    • 比较操作符（== < > != <> <= >= in not in is is not）
    • 逻辑操作符（and or not）
七、字符串

  • 单引号和双引号作用是一样的，但必须保证成对存在，不能夹杂使用

    （建议句子使用双引号，单词使用单引号，不强制）

八、注释
  • 注释块（通常应该在代码前，并和代码又同样的缩进）
  • 文档字符串
